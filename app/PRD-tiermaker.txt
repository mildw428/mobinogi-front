좋아요! 마비노기 모바일의 ‘룬’을 주제로 티어메이커(Tier Maker) 툴을 만들기 위한 \*\*제품 요구 사항 문서(PRD)\*\*를 아래에 작성해드릴게요. 이 문서는 기획자, 디자이너, 개발자 등 이해관계자 모두가 이해하기 쉬운 구조로 되어 있습니다.

---

## 📄 제품 요구 사항 문서 (PRD)

### 1. 제품명

**마비노기 모바일 룬 티어메이커**

---

### 2. 목적

마비노기 모바일 유저들이 게임 내 ‘룬’ 아이템들을 효율적으로 분류하고 공유할 수 있도록, 직관적인 UI를 갖춘 티어메이커 웹 툴을 제공한다.

---

### 3. 대상 사용자

* 마비노기 모바일 플레이어
* 룬 성능/활용도에 대한 정보를 알고 싶거나 공유하고 싶은 유저
* 커뮤니티(디스코드, 카페, 블로그 등)에서 토론/공유 목적으로 티어를 만들고자 하는 사용자

---

### 4. 핵심 기능

#### 4.1 룬 리스트 제공

* 마비노기 모바일에 존재하는 모든 룬의 아이콘, 이름, 기본 정보(속성, 능력 등)를 DB로 제공
* 정렬/검색 기능 포함 (이름, 속성, 등급 등)

#### 4.2 티어 슬롯 시스템

* 기본 티어 구조: S / A / B / C / D (커스터마이징 가능하게 옵션 제공)
* 사용자가 드래그 앤 드롭 방식으로 룬을 각 티어에 배치 가능
* 티어 이름 및 색상 사용자 지정 가능

#### 4.3 커스터마이징 기능

* 티어 이름 수정
* 티어 개수 조절 (최대 10개)
* 각 티어 배경 색상 지정
* 룬 설명 마우스 오버 시 팝업으로 표시

#### 4.4 저장 및 공유

* 로그인 없이도 이미지 저장 기능 제공 (PNG 다운로드)
* 로그인 시 사용자 계정에 티어 저장 가능 (Firebase/Google OAuth 등 고려)
* 생성한 티어를 공유 가능한 고유 URL 생성
* ‘공식 추천 티어’, ‘인기 티어’, ‘최신 티어’ 페이지 제공

#### 4.5 반응형 UI

* PC/모바일 모두 대응
* 드래그 앤 드롭 및 티어 구성 모바일 최적화 필요

---

### 5. 기술 사양

#### 프론트엔드

* React.js + TypeScript
* Drag & Drop 라이브러리: `react-beautiful-dnd` 또는 `dnd-kit`
* CSS 프레임워크: TailwindCSS

#### 백엔드 (선택 사항)

* Firebase (Authentication, Firestore, Hosting)
* 또는 Next.js API Routes + Supabase

#### 배포

* Vercel, Netlify 또는 Firebase Hosting

---

### 6. 데이터 스키마 예시 (룬)

| 필드명       | 타입     | 설명                |
| --------- | ------ | ----------------- |
| id        | string | 고유 ID             |
| name      | string | 룬 이름              |
| icon\_url | string | 룬 이미지 URL         |
| element   | string | 속성 (불, 물, 바람 등)   |
| rarity    | string | 등급 (일반, 희귀, 전설 등) |
| effect    | string | 주요 효과 요약          |

---

### 7. 향후 기능 확장 고려 사항

* 티어 간 비교 분석 (예: 다른 유저의 티어와 내 티어 비교)
* 룬 별 투표 및 댓글 기능
* 티어 기반 추천 룬 시스템
* 다른 게임 콘텐츠(예: 스킬, 무기 등)로의 확장 가능성

---

### 8. 일정 제안 (MVP 기준)

| 주차  | 항목                        |
| --- | ------------------------- |
| 1주차 | 기획안 작성, 룬 데이터 수집 및 정제     |
| 2주차 | 프론트엔드 UI/UX 프로토타입 제작      |
| 3주차 | 티어 배치 기능 구현 (Drag & Drop) |
| 4주차 | 티어 저장, 공유, 이미지 다운로드 기능    |
| 5주차 | 반응형 UI 적용 및 QA            |
| 6주차 | 오픈 베타 및 피드백 수렴            |
